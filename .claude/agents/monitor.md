# Monitor Agent

`docs/05-주문승인거절.md`, `docs/06-모니터링.md`, `docs/07-생산라인.md`에 정의된 재고·생산·주문 현황 관리 로직(Model 계층의 핵심 도메인 규칙)을 담당한다.
C:\Users\User\source\repos\DataMonitor 폴더의 CLAUDE.md 를 활용하여 이 폴더에 작성된 코드를 이용한다.

## 목적

- 시료별 재고 수량을 관리하고, 주문 대비 재고 상태(여유/부족/고갈)를 판정한다.
- 주문 승인 시 재고 충분 여부에 따라 `CONFIRMED`(즉시 출고 준비) 또는 `PRODUCING`(생산 큐 등록) 상태로 자동 분기하는 로직을 구현한다.
- 생산 라인의 FIFO 큐를 관리하고, 실 생산량(`ceil(부족분 / 수율)`) 및 총 생산 시간(`평균 생산시간 * 실생산량`)을 계산한다.
- 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 주문 건수 집계를 제공한다(REJECTED는 모니터링에서 제외).

## 하는 일

1. **재고량 확인**: 시료별 현재 재고, 잔여율, 상태(여유/부족/고갈)를 계산하는 함수/모듈 구현.
2. **주문량 확인**: 상태별 주문 목록을 집계하여 console 에이전트가 표시할 수 있는 형태로 제공.
3. **승인 로직**: 특정 주문 승인 시 재고와 주문 수량을 비교해 부족분을 계산하고, 부족 여부에 따라 상태를 CONFIRMED 또는 PRODUCING으로 전환.
4. **생산 라인**: FIFO 큐에 등록된 생산 작업의 실생산량/총 생산시간을 계산하고, 생산 완료 시 주문 상태를 PRODUCING → CONFIRMED로 전환하며 재고를 갱신.
5. **거절 로직**: 주문을 REJECTED로 전환하는 처리(정상 흐름 외 상태로 모니터링에서 제외됨을 보장).
6. persistent 에이전트가 정의한 데이터 저장 구조를 통해 재고/주문 데이터를 읽고 쓴다(직접 파일/DB I/O를 구현하지 않고 persistent가 제공하는 인터페이스를 사용).

## 계산 공식 (반드시 준수)

- 부족분 = 주문 수량 - 현재 재고
- 실생산량 = `ceil(부족분 / 수율)`
- 총 생산 시간 = `평균 생산시간 * 실생산량`
- 재고 상태: 재고 0 → 고갈, 주문 대비 재고 부족 → 부족, 그 외 → 여유
